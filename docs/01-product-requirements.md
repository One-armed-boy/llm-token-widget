# 제품 요구사항

## 목표

여러 Claude, Claude Code, Codex/OpenAI 계정의 사용량 상태를 macOS 위젯에서 즉시 판단할 수 있게 한다.

사용자가 알고 싶은 것은 세부 청구서가 아니라 "지금 어느 계정을 써도 되는가", "이번 달 한도에 가까운 계정은 무엇인가", "데이터가 얼마나 최신인가"입니다.

제품 포지션은 "안전하고 가벼운 다중 계정 위젯"입니다. 많은 provider를 깊게 지원하거나 브라우저 세션을 읽는 대신, 사용자가 명시적으로 제공한 공식 API credential과 수동 예산을 기반으로 신뢰 가능한 상태를 작게 보여줍니다.

## 1차 MVP

- macOS 메뉴바 앱
- macOS WidgetKit 위젯
- N개 계정 등록
- 계정별 상태등 표시: 정상, 주의, 위험, 알 수 없음
- 계정별 이번 달 사용량, 비용 또는 수동 예산 대비 상태, 마지막 갱신 시각
- 가장 안전하게 사용할 계정 1개 추천
- 수집 실패, 권한 부족, API 미지원 상태를 사용자가 구분 가능하게 표시
- privacy mode: 계정명/금액 숨김

## 비목표

- 채팅 클라이언트 구현
- 토큰 사용량 자체를 중간 프록시로 강제 계측하는 기능
- 공개 API가 없는 개인 구독 서비스의 비공식 스크래핑
- 브라우저 쿠키, WebView 로그인, 다른 앱의 OAuth token 재사용
- Full Disk Access가 필요한 자동 로그/쿠키 스캔
- 팀/조직용 상세 governance dashboard
- 공식 사용량과 로컬 로그 추정치의 reconciliation
- provider 대량 확장
- 초 단위 실시간 위젯 갱신 보장

## 주요 사용자

- Claude Code와 Codex를 동시에 쓰는 개인 개발자
- 여러 개인/업무 계정을 오가며 AI 코딩 도구를 쓰는 사용자
- 회사 Mac에서 브라우저 쿠키/Full Disk Access 없이 안전하게 사용량을 보고 싶은 사용자

## 핵심 UX

### 메뉴바

메뉴바 아이콘은 전체 상태를 하나의 색으로 요약합니다.

- 녹색: 모든 활성 계정 정상
- 노란색: 하나 이상 주의 또는 데이터 지연
- 빨간색: 하나 이상 한도 초과, 인증 실패, 또는 예산 위험
- 회색: 설정 필요 또는 수집 불가

클릭하면 계정 목록, 빠른 갱신, 설정 진입, 최근 오류를 보여줍니다.

### 위젯

위젯 크기별 정보 밀도를 다르게 합니다.

- Small: 추천 계정 1개 또는 가장 위험한 계정 1개, 마지막 갱신
- Medium: 추천 계정, 위험 계정, 계정 3-5개 요약
- Large: 계정별 상태 보드, 오류/권한 상태, privacy mode 표시

위젯 안에는 비밀값이나 긴 설명을 표시하지 않습니다. 사용자가 판단하는 데 필요한 숫자와 상태만 둡니다.

## 상태 판단 규칙

각 계정은 `remaining_percent`, `spend_percent`, `freshness`, `error_state`, `source_confidence`를 기준으로 상태를 계산합니다.

- 정상: 최신 데이터이고 사용률이 경고 임계값보다 낮음
- 주의: 사용률 70% 이상, 또는 데이터가 오래됨
- 위험: 사용률 90% 이상, 초과, 인증 실패, 권한 부족
- 알 수 없음: 공급자 API가 미지원이거나 필요한 권한이 없음

사용자 설정에서 임계값을 계정별로 조정할 수 있어야 합니다.

## 추천 규칙

MVP의 추천은 단순 규칙 기반입니다.

```text
recommendation_score =
  remaining_or_budget_score
  - stale_penalty
  - error_penalty
  - unsupported_penalty
  - manual_source_penalty
```

추천 결과에는 이유를 한 줄로 붙입니다.

예: `Claude Personal 권장: 월 예산 78% 남음, 4분 전 갱신`

## 성공 기준

- 등록된 5개 이상 계정의 상태를 위젯에서 3초 안에 판단할 수 있다.
- 위젯에서 추천 계정과 위험 계정을 구분할 수 있다.
- 수집 실패와 사용량 초과가 시각적으로 구분된다.
- 계정 추가 후 첫 스냅샷 생성까지 30초 이내를 목표로 한다.
- 메뉴바 앱에서 강제 갱신을 실행하면 성공/실패 원인을 확인할 수 있다.
- 브라우저 쿠키, WebView 로그인, Full Disk Access 없이 MVP가 동작한다.
